# AI识图比价 - 前后端接口文档

## 基础信息

| 项目 | 说明 |
|------|------|
| 后端地址 | `http://127.0.0.1:8000` |
| 接口前缀 | `/api` |
| 鉴权方式 | JWT (Header: `Authorization: Bearer <token>`) |
| 请求编码 | `Content-Type: application/json` |
| 统一返回格式 | `{"code": 200, "msg": "ok", "data": {}}` |

---

## 一、通用说明

### 1.1 统一返回格式

```json
{
  "code": 200,      // 200成功，其他为错误码
  "msg": "ok",      // 提示信息
  "data": {}        // 业务数据，失败时为 null
}
```

### 1.2 鉴权说明

| 接口类型 | Header要求 | 说明 |
|----------|-----------|------|
| 公开接口 | 无需携带 | 注册、登录 |
| 可选鉴权 | 可带可不带 | 比价接口（带token会保存历史） |
| 强制鉴权 | 必须携带 | 用户信息、历史记录等 |

Header格式：
```
Authorization: Bearer <token>
```

登录成功后，前端需将返回的 `token` 持久化存储（localStorage / cookie），每次请求在 Header 中携带。

## 三、比价核心模块

### 3.1 图片比价

拍照或相册选图后上传 base64 编码，AI识别商品并全网比价。

```
POST /api/compare/image
```

**请求头**
```
Authorization: Bearer <token>    // 选填（不填则不保存比价历史）
```

**请求体**
```json
{
  "image_base64": "string, 必填, 图片Base64编码字符串（不含data:image前缀）"
}
```

**前端调用示例**
```javascript
// 1. 选图后获取 base64
const file = e.target.files[0];
const reader = new FileReader();
reader.onload = () => {
  const base64 = reader.result.split(',')[1];  // 去掉 data:image/jpeg;base64, 前缀
  fetch('/api/compare/image', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${token}`
    },
    body: JSON.stringify({ image_base64: base64 })
  }).then(res => res.json()).then(data => {
    // data.data 即 CompareResult
  });
};
reader.readAsDataURL(file);
```

**成功响应** `code: 200` — 数据结构见 [3.4 比价结果通用结构](#34-比价结果通用结构)

### 3.4 比价结果通用结构

三种比价方式（图片/链接/关键词）返回结构**完全相同**，前端统一渲染。

```json
{
  "code": 200,
  "msg": "ok",
  "data": {
    "goods_info": "iPhone 16 Pro Max 256GB 黑色钛金属",
    "ai_analysis": {
      "price_level": "适中",
      "buy_advice": "当前价格处于近30天中低位，建议入手，预计未来一周波动不大",
      "price_prediction": "未来7天内价格预计小幅下降5-10元，建议观望3天后入手"
    },
    "price_history": [
      { "date": "2025-04-20", "price": 8899.00 },
      { "date": "2025-04-21", "price": 8799.00 },
      { "date": "2025-05-19", "price": 8569.00 }
    ],
    "compare_list": [
      {
        "goods_id": 101,
        "goods_name": "iPhone 16 Pro Max 256GB 黑色钛金属 5G全网通",
        "brand": "Apple",
        "model": "iPhone 16 Pro Max",
        "spec_param": "256GB 黑色钛金属",
        "platform": "拼多多",
        "shop_name": "百亿补贴官方",
        "original_price": 9999.00,
        "real_price": 8569.00,
        "coupon_desc": "百亿补贴满8000减1000",
        "goods_img": "https://img.alicdn.com/xxx.jpg",
        "goods_url": "https://mobile.yangkeduo.com/xxx",
        "sales_num": 28000,
        "score": 4.9,
        "sale_status": 1
      },
      {
        "goods_id": 102,
        "goods_name": "iPhone 16 Pro Max 256GB",
        "brand": "Apple",
        "model": "iPhone 16 Pro Max",
        "spec_param": "256GB 黑色",
        "platform": "淘宝",
        "shop_name": "天猫旗舰店",
        "original_price": 9999.00,
        "real_price": 8799.00,
        "coupon_desc": "",
        "goods_img": "https://img.alicdn.com/yyy.jpg",
        "goods_url": "https://detail.tmall.com/xxx",
        "sales_num": 52000,
        "score": 4.8,
        "sale_status": 1
      }
    ]
  }
}
```

**字段说明**

| 字段 | 类型 | 说明 |
|------|------|------|
| `data.goods_info` | string | AI识别的商品描述文本 |
| `data.ai_analysis.price_level` | string | 价位评价：偏高 / 适中 / 偏低 |
| `data.ai_analysis.buy_advice` | string | AI入手建议（50字内） |
| `data.ai_analysis.price_prediction` | string | 降价预测（50字内） |
| `data.price_history` | array | 30天历史价格走势 |
| `data.price_history[].date` | string | 日期 `YYYY-MM-DD` |
| `data.price_history[].price` | number | 当天价格（元） |
| `data.compare_list` | array | **已按 real_price 从低到高排序** |
| `data.compare_list[].goods_id` | int | 商品唯一ID |
| `data.compare_list[].platform` | string | 平台：淘宝 / 京东 / 拼多多 / 抖音 |
| `data.compare_list[].real_price` | number | 券后实际到手价 |
| `data.compare_list[].original_price` | number | 商品原价 |
| `data.compare_list[].sale_status` | int | **1=在售（正常显示）/ 0=下架（置灰）** |

**前端渲染规则**
1. `sale_status === 0` → 该商品卡片置灰 + 标注"已下架"
2. `compare_list` 已按 `real_price` 升序排列，直接渲染即可
3. `real_price` 为到手价，应突出显示；`coupon_desc` 有值时展示优惠券标签
4. `price_history` 可用于折线图（推荐 ECharts / Chart.js）
5. `ai_analysis` 三个字段分别对应：价位标签、入手建议卡片、降价预测卡片

## 四、历史记录模块

### 4.1 获取比价历史列表

```
GET /api/record/list
```

**请求头**
```
Authorization: Bearer <token>    // 必填
```

**成功响应** `code: 200`
```json
{
  "code": 200,
  "msg": "ok",
  "data": [
    {
      "id": 1,
      "search_source": "iPhone 16 Pro Max",
      "compare_type": 3,                      // 1=图片, 2=链接, 3=关键词
      "create_time": "2025-05-19 10:30:00"
    },
    {
      "id": 2,
      "search_source": "https://detail.tmall.com/xxx",
      "compare_type": 2,
      "create_time": "2025-05-18 15:20:00"
    }
  ]
}
```

**字段说明**
| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | int | 记录ID，用于删除 |
| `search_source` | string | 比价源头内容 |
| `compare_type` | int | 1=图片比价, 2=链接比价, 3=关键词比价 |
| `create_time` | string | 创建时间 `YYYY-MM-DD HH:mm:ss` |

**前端逻辑**
- 用户点击某条历史 → 根据 `compare_type` 调用对应比价接口重新比价：
  - `compare_type=1`: 因图片 base64 过大，历史中 `search_source` 存的是AI识别后的描述文本，调用关键词比价接口
  - `compare_type=2`: 调用 `POST /api/compare/url` { goods_url: search_source }
  - `compare_type=3`: 调用 `GET /api/compare/search?keyword=search_source`

## 五、商品模块

### 5.1 获取商品历史价格

查询单个商品近30天价格走势，用于绘制折线图。

```
GET /api/goods/price/history?goods_id=101
```

**Query参数**
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| goods_id | int | 是 | 商品ID |

**成功响应** `code: 200`
```json
{
  "code": 200,
  "msg": "ok",
  "data": {
    "goods_id": 101,
    "price_history": [
      { "date": "2025-04-20", "price": 8899.00 },
      { "date": "2025-04-21", "price": 8799.00 },
      { "date": "2025-05-19", "price": 8569.00 }
    ]
  }
}
```

**注意**：比价结果中的 `price_history` 是所有同款商品的合并价格走势，本接口返回的是**单个商品**的历史价格。

## 六、前端对接指南

### 6.1 请求封装示例（JavaScript）

```javascript
const BASE_URL = 'http://127.0.0.1:8000';
const TOKEN_KEY = 'auth_token';

// 通用请求函数
async function request(method, path, { params, body, auth = true } = {}) {
  const headers = { 'Content-Type': 'application/json' };

  if (auth) {
    const token = localStorage.getItem(TOKEN_KEY);
    if (token) headers['Authorization'] = `Bearer ${token}`;
  }

  let url = `${BASE_URL}${path}`;
  if (params) url += '?' + new URLSearchParams(params);

  const res = await fetch(url, {
    method,
    headers,
    body: body ? JSON.stringify(body) : undefined,
  });

  const json = await res.json();

  if (json.code === 401) {
    localStorage.removeItem(TOKEN_KEY);
    window.location.hash = '#/login';
    throw new Error('请先登录');
  }

  if (json.code !== 200) {
    throw new Error(json.msg);
  }

  return json.data;
}

// 使用示例
const userInfo = await request('GET', '/api/user/info');
await request('POST', '/api/user/register', { body: { username, password, phone }, auth: false });
```

### 6.2 页面 ↔ 接口映射

| 页面 | 调用接口 |
|------|----------|
| **登录注册页** | `POST /api/user/login`, `POST /api/user/register` |
| **首页** | 三个入口分别跳转对应页面 |
| **拍照/相册选择页** | 选图 → base64 → `POST /api/compare/image` → 跳转结果页 |
| **链接输入** | 粘贴链接 → `POST /api/compare/url` → 跳转结果页 |
| **关键词搜索** | 输入文字 → `GET /api/compare/search` → 跳转结果页 |
| **比价结果页** | 渲染 `ai_analysis` + `price_history`折线图 + `compare_list`列表 |
| **比价历史页** | `GET /api/record/list` 获取列表；点击条目 → 重新调用对应比价接口 |
| **个人中心页** | `GET /api/user/info` 展示；`PUT /api/user/update` 修改 |

### 6.3 比价结果页渲染要点

```
┌─────────────────────────────────┐
│  [返回]      商品比价结果         │
├─────────────────────────────────┤
│  ┌─ AI分析卡片 ────────────────┐ │
│  │ 识别商品：goods_info         │ │
│  │ 价位评价：[price_level 标签] │ │
│  │ 入手建议：buy_advice         │ │
│  │ 降价预测：price_prediction   │ │
│  └────────────────────────────┘ │
│  ┌─ 30天价格走势 ──────────────┐ │
│  │  [折线图：price_history]     │ │
│  └────────────────────────────┘ │
│  ┌─ 全网比价列表 ──────────────┐ │
│  │  compare_list 每项一个卡片    │ │
│  │  按real_price升序                     │ │
│  │  sale_status=0 → 置灰        │ │
│  └────────────────────────────┘ │
└─────────────────────────────────┘
```

### 6.4 状态码速查

| code | 含义 | 处理方式 |
|------|------|----------|
| 200 | 成功 | 正常处理 data |
| 400 | 参数/业务错误 | toast提示 msg |
| 401 | 未登录/token过期 | 跳转登录页 |
| 404 | 资源不存在 | toast提示 msg |
| 500 | 服务端异常 | toast提示 msg |
