请完整开发一套 AI识图比价APP，分为【后端FastAPI项目】+【移动端UI页面设计】，严格按照下面需求开发，包含：数据库设计、接口设计、UI页面、功能逻辑、业务流程。

=== 一、项目技术栈 ===
后端：Python FastAPI
数据库：MySQL 8.0
向量库：Milvus
鉴权：JWT
AI能力：图片识别、商品同款筛选、价格分析、入手建议
缓存：Redis
架构：企业级前置爬虫架构，后台定时爬虫，用户只做查询，不实时爬取。

=== 二、功能需求（详细） ===
1. 用户模块：手机号注册、登录、JWT鉴权、个人中心、退出登录。
2. 首页三大比价入口：
   （1）拍照识图比价：拍照/相册选图，AI识别商品，全网比价
   （2）链接比价：粘贴淘宝/京东/拼多多/抖音商品链接比价
   （3）关键词搜索比价：输入商品名称搜索比价
3. 比价结果页：
   - 展示识别的商品信息
   - AI入手建议（当前价位高低、是否可买、降价预测）
   - 30天历史价格走势图
   - 全网同款商品列表，按到手价从低到高排序，显示平台、价格、店铺、跳转购买
   - 下架商品置灰标注
4. 比价历史：只存储搜索源头（图片/链接/关键词），点击历史重新比价，获取最新数据。
5. 后台自动定时爬虫采集商品，存入MySQL，价格变动自动存入历史价格表，商品特征向量化存入Milvus。
6. 比价流程：AI提取信息→生成查询向量→Milvus检索相似goods_id→批量查MySQL在售商品+历史价→AI筛选同款→返回前端。

=== 三、MySQL数据库（4张表，严格按字段） ===
CREATE TABLE `user` (
  `id` bigint NOT NULL AUTO_INCREMENT COMMENT '用户主键ID',
  `username` varchar(32) NOT NULL COMMENT '用户名',
  `password` varchar(64) NOT NULL COMMENT '加密密码',
  `phone` varchar(11) DEFAULT '' COMMENT '手机号',
  `avatar` varchar(512) DEFAULT '' COMMENT '用户头像地址',
  `status` tinyint DEFAULT 1 COMMENT '1正常 0禁用',
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP,
  `update_time` datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  INDEX idx_phone(phone)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';

CREATE TABLE `goods` (
  `id` bigint NOT NULL AUTO_INCREMENT COMMENT '商品唯一ID，关联向量库',
  `goods_name` varchar(255) NOT NULL COMMENT '商品全称',
  `brand` varchar(64) DEFAULT '' COMMENT '品牌名称',
  `model` varchar(128) DEFAULT '' COMMENT '商品型号',
  `spec_param` varchar(255) DEFAULT '' COMMENT '规格配置',
  `platform` varchar(32) NOT NULL COMMENT '平台：淘宝/京东/拼多多/抖音',
  `shop_name` varchar(128) DEFAULT '' COMMENT '店铺名称',
  `original_price` decimal(10,2) DEFAULT 0.00 COMMENT '原价',
  `real_price` decimal(10,2) DEFAULT 0.00 COMMENT '券后真实到手价',
  `coupon_desc` varchar(512) DEFAULT '' COMMENT '优惠券说明',
  `goods_img` varchar(512) DEFAULT '' COMMENT '商品主图链接',
  `goods_url` varchar(800) NOT NULL COMMENT '商品原生购买链接',
  `sales_num` int DEFAULT 0 COMMENT '销量',
  `score` decimal(3,1) DEFAULT 0.0 COMMENT '商品评分',
  `feature_content` text COMMENT '商品特征文本，用于生成向量',
  `sale_status` tinyint DEFAULT 1 COMMENT '1在售 0下架',
  `last_update_time` datetime DEFAULT CURRENT_TIMESTAMP,
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  INDEX idx_platform(platform),
  INDEX idx_brand(brand),
  INDEX idx_sale_status(sale_status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='全网商品主表';

CREATE TABLE `goods_price_history` (
  `id` bigint NOT NULL AUTO_INCREMENT,
  `goods_id` bigint NOT NULL COMMENT '关联商品主表ID',
  `price` decimal(10,2) NOT NULL COMMENT '记录价格',
  `record_date` date NOT NULL COMMENT '价格记录日期',
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  INDEX idx_goods_id(goods_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='商品历史价格记录表';

CREATE TABLE `user_compare_record` (
  `id` bigint NOT NULL AUTO_INCREMENT,
  `user_id` bigint NOT NULL COMMENT '关联用户ID',
  `search_source` varchar(1000) NOT NULL COMMENT '比价源头：链接/识图文本/关键词',
  `compare_type` tinyint NOT NULL COMMENT '1图片 2链接 3关键词',
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  INDEX idx_user_id(user_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户比价历史记录表';

=== 四、Milvus向量库关联规则 ===
1. 唯一关联键：goods.id
2. 爬虫新增商品：插入MySQL后获取自增ID，生成向量存入Milvus
3. Milvus字段：vector向量、goods_id、brand、platform
4. 价格更新只改MySQL，不修改向量；款式变化更新向量
5. 下架商品sale_status=0，SQL查询自动过滤

=== 五、接口设计（RESTful，统一返回格式） ===
统一返回格式：{"code":200,"msg":"","data":{}}

1. 用户模块
POST /api/user/register 注册
POST /api/user/login 登录返回JWT
GET /api/user/info 获取用户信息
PUT /api/user/update 修改信息

2. 比价核心接口
POST /api/compare/image 图片比价，入参image_base64、token
POST /api/compare/url 链接比价，入参goods_url、token
GET /api/compare/search 关键词比价，入参keyword、token

3. 历史记录接口
GET /api/record/list 获取用户比价历史
DELETE /api/record/delete 删除单条历史

4. 商品接口
GET /api/goods/price/history 获取商品历史价格

5. 内部接口
POST /api/spider/add_goods 爬虫入库
POST /api/vec/insert 向量入库
POST /api/vec/search 向量检索

=== 七、业务流程 ===
1. 后台定时爬虫→入库MySQL→价格变动写入历史表→生成向量存入Milvus
2. 用户比价：AI解析→生成查询向量→Milvus召回goods_id→查MySQL在售商品+历史价→AI筛选同款→返回前端
3. 保存比价源头到历史表
4. 查看历史：读取源头→重新比价→展示最新结果
