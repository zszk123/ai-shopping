# AI 比价系统 UI 重设计说明

本方案只做 UI/UX 设计，不修改现有项目代码。设计方向来自 `ui-ux-pro-max` 检索结果，并结合当前项目的 Flutter 移动端形态、图片比价、结果页、AI 客服、历史记录等核心业务。

## 设计定位

产品定位：AI 电商比价助手。

目标体验：快、可信、清楚。用户上传图片或输入商品后，系统应快速告诉用户“识别到什么、最低可信价是多少、为什么推荐、下一步做什么”。

设计风格：Flat Design Mobile，触控优先、低阴影、清晰分区、信息密度适中。

结构：

```text

密码输入框
登录按钮
手机号验证码登录
登录/注册切换
```

### 首页

核心目标：让用户一眼知道可以用三种方式比价。

结构：

```text
顶部深色品牌区
搜索框
拍照识图主入口
链接比价 / 关键词比价次入口
推荐工作流说明
底部导航
```

优化点：

```text
拍照识图是最高频入口，做成第一屏主 CTA
链接和关键词作为次级入口
用“今日推荐工作流”解释系统可信度来源
```

### 比价结果页

核心目标：减少用户理解成本，让用户知道“该不该买”。

结构：

```text
识别商品卡片
AI 购买建议卡片
价格趋势卡片
全网同款列表
```

优化点：

```text
先给结论，再给数据
列表中突出最低价、平台、销量、可信分
空结果时显示“暂未找到同款商品”并提供手动输入入口
```

### AI 客服页

核心目标：让客服 Agent 体现“能调用工具”，不是普通聊天框。

结构：

```text
客服对话流
快捷问题
图片按钮 + 输入框 + 发送按钮
工具调用摘要
```

优化点：

```text
支持图片咨询
支持商品比价、历史查询、运行排查
回复里展示识别结果、候选商品数、价格建议
```

### 历史页

核心目标：让历史记录能继续触发比价，而不只是列表。

结构：

```text
历史列表
每条记录显示来源、类型、时间
月度省钱估算卡片
体验规则说明
```

优化点：

```text
点击历史记录可重新比价
显示“本月帮你省钱”增强产品价值感
```

## 失败页与空状态设计

失败状态不能只显示错误文案，必须给用户恢复路径。单独设计板：

```text
C:\Users\吴蒙蒙\Desktop\ai-shopping\design\ai-shopping-failure-states-board.svg
```

### 图片比价失败

适用场景：

```text
图片模糊
商品主体太小
图片不是商品图
AI 没有提取到 goods_name / brand / model / specs
```

页面文案：

```text
标题：没有识别到有效商品
说明：可能是图片模糊、主体太小、截图信息不完整，或当前图片不是商品图。
```

操作：

```text
主按钮：重新上传图片
次按钮：关键词比价
次按钮：链接比价
辅助说明：商品主体占画面 60% 以上；避免反光、遮挡、过暗；截图尽量包含商品标题或型号。
```

### AI 服务异常

适用场景：

```text
DashScope AccessDenied
模型限流
模型超时
OSS 上传失败
AI 返回内容无法解析
```

页面文案：

```text
标题：AI 识图暂时不可用
说明：可能是模型权限、服务限流或网络波动。你可以先用文字方式继续比价。
```

操作：

```text
主按钮：再次重试
次按钮：改用关键词比价
次按钮：粘贴商品链接
开发者信息：折叠展示错误码，不直接吓到普通用户
```

### 后端或网络连接失败

适用场景：

```text
后端没有启动
端口 8000 不通
CORS / API_BASE_URL 配置错误
readyz 不是 ready
```

页面文案：

```text
标题：暂时连不上后端服务
说明：请确认后端已启动，并且 readyz 状态正常。
```

操作：

```text
主按钮：重新连接
次按钮：查看排查指南
检查顺序：readyz -> database/redis/milvus -> API_BASE_URL
```

### 没有同款商品

适用场景：

```text
图片识别成功
关键词也生成成功
但向量库或 MySQL 没找到足够相似的商品
```

页面文案：

```text
标题：暂未找到同款商品
说明：识别成功了，但商品库里没有足够相似的在售商品。
```

操作：

```text
主按钮：补充品牌型号搜索
次按钮：换张图片
次按钮：链接比价
推荐关键词：品牌 + 型号 / 容量 / 颜色
```

### 登录失效或无权限

适用场景：

```text
查看历史记录
AI 客服查询用户历史
个人中心接口 401
```

页面文案：

```text
标题：登录后继续使用
说明：当前登录已失效，重新登录后可以继续使用图片比价、历史记录和 AI 客服。
```

操作：

```text
主按钮：去登录
次按钮：找回密码
```

### 设计原则

```text
失败页必须有明确标题
失败页必须有一句用户能听懂的原因
失败页必须有一个主恢复按钮
失败页至少提供一个替代路径
技术错误码只能作为折叠信息展示
不要出现纯白屏、纯错误文本、无按钮的失败状态
```
