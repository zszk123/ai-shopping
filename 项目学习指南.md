# AI 商品比价助手项目学习指南

这份文档是给初学者看的。目标不是一次讲完所有细节，而是帮你知道：这个项目由哪些知识组成，每个知识在项目里怎么用，应该从哪里开始读代码。

## 1. 项目整体在做什么

这个项目是一个 AI 商品比价助手，分成两部分：

- `frontend`：Flutter 前端，负责页面、拍照、输入链接/关键词、展示比价结果。
- `backend`：FastAPI 后端，负责登录注册、调用 AI、查数据库、查向量库、返回比价结果。

核心流程是：

```text
用户输入商品信息
  -> 前端调用后端接口
  -> 后端用 AI 识别/提取商品关键词
  -> 生成向量
  -> 去 Milvus 找相似商品
  -> 去 MySQL 查商品价格
  -> 用 AI 重排同款商品
  -> 生成价格分析
  -> 返回给 Flutter 展示
```

### Riverpod 是什么

Riverpod 是 Flutter 的状态管理工具。它负责把数据和页面连接起来。

项目里常见写法：

```dart
final compareProvider = StateNotifierProvider<CompareNotifier, CompareState>((ref) {
  final api = ref.watch(apiClientProvider);

```

你可以理解成：

```text
Provider = 数据仓库
Notifier = 修改数据的人
State = 当前数据
页面 watch State，数据变化后自动刷新
```

使用方式：

```dart
final compareState = ref.watch(compareProvider);
ref.read(compareProvider.notifier).compareByKeyword(keyword);
```

第一个是读取状态，第二个是调用方法修改状态。

## 6. Milvus 向量库

Milvus 是向量数据库。它不是按普通文字查，而是按“语义相似度”查。

例如用户搜：

```text
iPhone 16 Pro Max 256GB
```

系统会生成一个向量，再去 Milvus 查语义上相似的商品。

项目位置：

```text
app/services/vector_service.py
```

核心方法：

```python
search_similar_goods(query_vector, top_k=50)
```

返回的是相似商品的 `goods_id`，后面再去 MySQL 查完整商品信息。

用户上传图片后，后端不能直接把本地文件给 AI 模型看，所以先上传到 OSS，得到一个图片 URL。



```text

  -> 前端保存 token
  -> 后续请求带上 Authorization: Bearer token

```

项目位置：


## 14. 常见问题

### 为什么要用 Milvus，不直接 MySQL 搜索？

MySQL 适合精确查询，比如按 ID、平台、价格查。

Milvus 适合语义相似查询，比如：

```text
苹果手机 256G
iPhone 16 Pro Max 256GB
```

这两个文字不完全一样，但语义很接近。

### 为什么要先 Milvus 再 MySQL？

Milvus 只负责找相似商品 ID。

MySQL 保存完整商品信息，比如价格、店铺、销量、图片。

所以流程是：

```text
Milvus 找 goods_id
MySQL 根据 goods_id 查完整商品
```

### 为什么前端要有 Model？

因为后端返回的是 JSON，直接用字符串容易乱。

Model 可以把 JSON 变成有类型的对象，让代码更安全。

### 为什么要有 Service 层？

Router 只负责接收请求。

Service 负责业务逻辑。

这样代码更清晰，也更容易测试。
