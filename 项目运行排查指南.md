# 项目运行排查指南

这份文档按“小白也能照着查”的顺序写。遇到页面报错时，先不要反复重启，按下面顺序看。

## 1. 先确认后端是不是唯一运行

在 PowerShell 执行：

```powershell
netstat -ano | findstr :8000
```

正常情况只应该看到一行 `LISTENING`。如果看到多个监听，说明开了多个后端，必须只保留一个。

如果只看到 `TIME_WAIT`，这不是正在运行的服务，等几十秒或直接重新启动后端即可。

## 2. 后端启动命令

进入后端目录：

```powershell
cd C:\Users\吴蒙蒙\Desktop\ai-shopping\backend
```

建议用项目虚拟环境启动：

```powershell
.\.venv\Scripts\activate
uvicorn app.main:app --reload
```

看到 `Application startup complete.` 表示进程启动完成，但还要继续看第 3 步。

注意：如果自检脚本显示 `Python executable` 是 `E:\anaconda\python.exe`，说明 PyCharm 当前可能没有用项目里的 `.venv`。这不一定立刻报错，但容易出现依赖版本和终端不一致的问题。建议 PyCharm 的 Python Interpreter 选择：

```text
C:\Users\吴蒙蒙\Desktop\ai-shopping\backend\.venv\Scripts\python.exe
```

## 3. 检查后端是否真正可用

打开浏览器访问：

```text
http://127.0.0.1:8000/healthz
http://127.0.0.1:8000/readyz
```

`healthz` 表示后端进程活着。

`readyz` 表示 MySQL、Redis、Milvus 都能连上。只有 `readyz` 里 `database`、`redis`、`milvus` 都是 `true`，才算完整可用。

也可以执行自检脚本：

```powershell
python scripts/dev_doctor.py
```

## 5. 常见报错含义

`WinError 10048`：8000 端口已经被占用。通常是上一个后端没关干净，或者 PyCharm 又开了一个后端。

`TIME_WAIT`：不是后端进程占用端口，是 Windows 网络连接的短暂等待状态，一般不用杀。

`database=true/false 交替出现`：这是之前数据库连接池预检和 `aiomysql` 的兼容问题，已经改成稳定的 `SELECT 1` 检查加连接回收。改完后需要重启后端才生效。

页面提示“后端连接失败”：可能是后端没启动，也可能是后端返回了错误。现在前端会尽量显示后端真实错误。

`/readyz` 里 `database=false`：MySQL 没连上，检查 `.env` 里的 MySQL 地址、端口、库名。

`/readyz` 里 `milvus=false`：Milvus 没连上，检查 Milvus 是否启动、`.env` 是否是 `MILVUS_HOST=127.0.0.1` 和 `MILVUS_PORT=19530`。

`/readyz` 里 `redis=false`：Redis 没连上。现在 Redis 掉线不会让核心比价直接失败，但登录限流和缓存会不可用。

## 6. 本次已修复的运行坑

Redis 从强依赖改成可降级：Redis 挂了不会直接让普通接口崩掉。

修复 MySQL readiness 偶发失败：去掉会触发 `aiomysql ping() missing reconnect` 的连接池预检，并在后端关闭时释放数据库连接池。

向量库或 Embedding 失败时，比价会自动退回 MySQL 关键词搜索，避免 Milvus 临时异常导致整个比价不可用。

开发环境默认关闭自动爬虫调度器，避免后端启动时后台爬虫干扰排错。

图片比价页面增加了状态清理和重试保护，浏览器返回后再次比价不应该复用上一次的坏状态。

前端接口错误解析已优化，登录和比价失败时会尽量显示后端真实原因，而不是笼统显示网络失败。

## 8. AI 客服 Agent

项目现在新增了一个旁路 AI 客服入口，不影响原来的图片比价、链接比价、关键词比价接口。

后端接口：

```text
POST /api/agent/chat
POST /api/agent/chat/image
```

前端入口：

```text
底部导航 -> AI客服
```

这个客服不是简单聊天框，而是轻量工具编排：

```text
用户问题
  ↓
判断意图：比价 / 历史 / 运行排查 / 普通咨询
  ↓
按需调用工具：关键词比价、用户历史、故障诊断
  ↓
整理成客服式回复
```

注意：客服 Agent 是旁路能力。它复用已有服务，但不修改原有比价主流程，所以原业务稳定性不受影响。

客服也支持上传商品图片。前端在 AI 客服输入框左侧有图片按钮，选择图片后会调用图片识别和比价工具，然后用客服口吻总结识别结果、候选商品和购买建议。
